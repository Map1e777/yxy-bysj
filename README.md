# 校园通勤班车智能排班系统

重庆邮电大学本科毕业设计（2022214430 袁湘洋）

## 技术栈说明

| 层次 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + Vite + Vue Router + Pinia + Axios + Element Plus + ECharts（Leaflet 地图） |
| 后端 | Node.js + Express |
| 数据库 | MySQL 8.0 |
| 算法 | 遗传算法（GA）+ 粒子群优化（PSO）混合排班 |

> **技术路线说明**：开题报告原计划采用 Spring Boot 后端，后续综合考虑接口开发效率、轻量化部署成本和团队熟悉度，调整为 Node.js + Express，两者均属主流 RESTful 后端框架，核心业务逻辑不变。

## 已实现功能模块

### 管理端
- **运行概览仪表盘**：关键 KPI 卡片 + 实时预警
- **实时监控地图**：Leaflet 地图展示线路轨迹、站点位置、车辆模拟实时位置与状态
- **站点地图定位**（管理员）：在地图上拖动或点击设置站点真实坐标，坐标写入数据库
- **智能排班**：GA + PSO 混合算法自动生成班次，三候选方案评分对比，一键发布
- **排班管理**：手动新增/编辑班次、批量状态调整（发布/取消）、操作日志、版本快照与回滚
- **调度事件**：事件登记、执行、回滚、通知推送完整闭环
- **统计分析**：线路热度、满载率分布、事件等级、空驶率分析（含基线对比与排班准确率）
- **基础资源管理**：线路、站点、车辆、驾驶员（驾照/工时约束）、通知管理
- **数据管理**：客流录入、道路状态管理、CSV 批量导入/导出、用户管理
- **系统参数**：算法权重配置（需求匹配 / 空驶率 / 等待时间 / 调度成本）

### 师生端
- 班次查询、线路收藏、历史意见反馈、实时车辆位置查询

## 快速启动

### 1. 数据库初始化

```bash
# 方式一（推荐）：使用初始化脚本
bash scripts/init-db.sh

# 方式二：在 MySQL 中直接执行
mysql -u root -p campus_shuttle < sql/init.sql
```

### 2. 后端启动

```bash
cd backend
cp .env.example .env      # 填写数据库配置
npm install
npm run dev
```

后端接口地址：`http://localhost:3000/api`

### 3. 前端启动

```bash
cd frontend
npm install
npm run dev
```

前端访问地址：`http://localhost:5173`

## 算法设计简述

智能排班采用 **GA + PSO 两阶段混合策略**：

1. **遗传算法（GA）**：种群 20 个体，迭代 30 代，锦标赛选择 + 单点交叉 + 15% 变异率，精英保留
2. **粒子群优化（PSO）**：5 粒子，10 轮局部搜索，对 GA 最优解做连续空间精化
3. **适应度函数**：`F = 0.4×需求匹配 + 0.3×空驶控制 + 0.2×等待时长 + 0.1×调度成本`（权重可在系统参数页配置）
4. **输出**：三候选方案（均衡 / 高峰优先 / 效率优先），自动选取综合分最高方案落库

## 核心性能指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 排班适配准确率 | ≥ 90% | 有客流需求的时段被已发布班次覆盖的比例 |
| 智能排班生成耗时 | ≤ 30 秒 | 含 GA + PSO 全流程 |
| 紧急调度响应 | ≤ 5 分钟 | 事件登记到班次调整完成 |
| 空驶率降低 | 10%–25% | 相较固定间隔基线方案 |
| 平均等待时间 | ≤ 8 分钟 | 高峰时段 |
